# Security

## Scope

The project only processes manifests and subtitle resources. It must not bypass DRM, account authorization, subscriptions, geo-entitlement, or playback licensing.

## API keys

Provider secrets are stored under a separate `$persistentStore` key and are not returned by `/health` or `/api/config`. They are not encrypted by General Stream Subtitle and remain recoverable by software or users with access to the proxy client's data.

Use restricted, low-limit, revocable keys. For high-value LLM keys, prefer a self-hosted relay that applies authentication, rate limits, model allowlists, and spending controls.

## Logs

Never publish:

- API keys or Authorization headers;
- account cookies;
- signed playback URLs;
- MITM private keys or exported CA identities;
- device identifiers;
- provider billing or project identifiers when they are sensitive.

## Local configuration page

The page at `gss.local` is a synthetic response generated by the request script. Write actions require a per-installation token. It is not a real listening web server and does not provide protection against a malicious local proxy configuration.
