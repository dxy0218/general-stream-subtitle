# Security

- Never publish MITM private keys, exported certificates, account cookies, Authorization headers, signed playback URLs or device identifiers.
- The project only processes manifests and subtitle text. It must not bypass DRM or account authorization.
- The local configuration page uses a persistent per-installation token for write actions. It is a synthetic response generated by the proxy script, not a real background web server.
- Report security issues privately to the repository owner before opening a public issue.
